v0.60 — Math reference, splash refresh & a documented bug audit

- **New `docs/FORMULAS.md` — one traceable home for every formula.** Scoring (line-cutter effective distance, multi-spot nearest-center, WA ring builder, quiver/end/set-play tallies), the `/analyze` shot-cloud statistics (centroid/MPI, σ_x/σ_y, ρ, the Rayleigh vs. bivariate-normal CEP/R95 derivations and their χ²₂ constants), the head-to-head test stack (Hotelling's T², Brown-Forsythe, Mann-Whitney U + Cliff's δ, Holm-Bonferroni, and the scipy-free numerics behind them), and the prediction path (server `_report_expected_score` Monte-Carlo with the empirical-miss blend, and the client `apollo-predict.js` mirror) are each written out next to the function that implements them. Every formula was re-derived from the code while writing — including a check that the expected-score miss blend reduces exactly to an `m`-vs-`(1−m)` mixture of "guaranteed miss" and "Gaussian sample." Linked from the README and from `documentation/tournament/scoring.md`.

- **README accuracy fix.** The "Tournament mode" note listed rounds as "WA / NFAA / USAA / NASP," but NASP no longer has a round in the selector (only World Archery, NFAA, and USA Archery rounds are defined). Corrected the list, called out that the NASP 40 cm face ships as the *default seeded target* rather than a round, and noted live match play.

- **Splash page brought back in sync with the app.** The Analytics section still advertised "six built-in reports" and "Welch's t-test, Cohen's d" — both long superseded. It now describes the accuracy-vs-precision split, the full current report suite (trends over time, within-session drift, cold-bore vs. warmed-up, draw-weight tuning, shot-density and calendar heatmaps, expected-score projection), and the real head-to-head statistics (Hotelling's T² / Brown-Forsythe / Mann-Whitney U + Cliff's δ, Holm-Bonferroni corrected). The Tournament section now mentions live match play (2–4 archers on one device).

- **Bug audit: clean bill of health.** A full sweep of the scoring, persistence, auth/CSRF, and import paths found no high- or medium-severity bugs: SQL is parameterized with validated identifiers, divisions are guarded, sessions/CSRF/rate-limiting are sound, and the import path has zip-slip / path-traversal / decompression-bomb defenses. One flagged "possible `min()` crash on empty `spot_centers_mm`" was confirmed a non-issue (an empty list is falsy, so the single-spot branch already handles it).

- **Minor defensive polish (behavior-neutral).** Switched the remaining positional `_shot_is_x(...)` call in `_compute_tournament_progress` to keyword arguments so the shaft-diameter and spot-centers args can't drift; documented the warm→cool color-ramp fallback in `_sample_zone_colors`; and added a one-line docstring to `_median`.

- **Install verification.** Re-checked `install.py` / `install_help.txt` against the environment variables `apollo.py` actually reads — all eleven (`SECRET_KEY`, `APOLLO_BACKEND`, `DATABASE_URL`, `APOLLO_BASE_URL`, `APOLLO_ROOT_USERNAME/EMAIL/PASSWORD`, `RESEND_API_KEY`, `RESEND_FROM`, `FLASK_ENV`, `FLASK_DEBUG`) match. No changes needed.
