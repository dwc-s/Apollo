v0.54 — Per-session draw weight · two new analyze reports · mobile polish

- **Effective draw weight is now a per-session value.** Effective DW (what you're actually pulling, vs. the bow's rated spec) changes every time the string goes back on the bow, so attaching it to the bow row left it stale. New "Effective draw weight (lbs, optional)" input on the session page (`/sesh`) and tournament page (`/tournament`); same field added to Edit session. The field auto-prefills from the most recent value the archer recorded for the selected bow, so it sticks across sessions without locking the user in — switching bows updates the prefill. The corresponding fields were removed from Add bow and Edit bows. `bows.effective_draw_weight` is left in place for historical data; new shots snapshot the per-session value onto the apollo row, and analysis (`_shot_effective_draw_weight`) still prefers the snapshot, falling back to the rated DW.

- **New analyze report — accuracy & precision traces.** Single chart with up to six chronological traces overlaid on one date axis: per-session accuracy (MPI) and precision (R95), per-quiver accuracy and precision (using the same completed-quiver slicer as the existing drift report), and an all-time rolling pool. Red traces are accuracy; green traces are precision. Values normalized by target half-width so mixed-target histories are comparable; misses excluded. Accepts a date range.

- **New analyze report — accuracy & precision by draw weight.** X-axis is draw weight (lb); every bow contributes up to four traces — accuracy (MPI, circle/triangle markers) and precision (R95, square/diamond markers), split by whether the session was shot at the bow's rated DW (dashed) or at a different effective DW (solid). A session is classed "effective" when its snapshotted `effective_draw_weight` differs from the rated `bow_draw_weight` by more than 0.05 lb. Each bow gets a distinct hue. Useful for telling whether re-tuning to a different effective draw weight actually changed how the bow shoots.

- **Tags more resilient between quivers (especially on mobile).** The chip input now mirrors any in-progress draft text into the hidden form input on every keystroke — defends against mobile blur/submit races where the user taps a button before the chip commits. Recall arrow now stashes the current tag string in `sessionStorage` before reloading; if the recall removed the only shot carrying tags, the chip widget is re-initialized with the stashed value so the input doesn't blank.

- **Mobile page titles no longer get clipped.** Long titles like "Previous sessions" and "Tournament setup" were being clipped at narrow widths because the `.title` rule used `white-space: nowrap; overflow: hidden;` and the mobile clamp couldn't shrink the font enough. Mobile rule now allows wrapping (`white-space: normal; overflow: visible`) with a lower clamp floor (`clamp(28px, 11vw, 72px)`) and a tighter line-height. Verified at 320px width: titles wrap to a second line cleanly with no clipping.

- **Previous sessions: Notes and Tags labels.** Session details now render `Notes: <text>` and `Tags: <chips>` with a colon and space between the label and the value, instead of running them together.
