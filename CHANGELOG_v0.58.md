v0.58 — Live match play · CSRF token longevity

- **Live match play — 2–4 archers take turns on one device.** Each tournament round now offers a third path, **"Match play"**, alongside Practice and the paper scoresheet. Enter 2–4 archers by name and they share the canvas: an archer shoots a full end, then the target hands off to the next archer (AB-CD "detail" shooting, WA Book 3 §11.1.1), cycling back to the first archer for the next end. Each archer gets their own `session_id`, all grouped under one `match:<id>` tag (reusing the scoresheet flow's multi-participant model), so every shot is recoverable and the existing match-results page works unchanged. The active archer's session is mirrored into the session cookie, so the whole single-archer render path (canvas, past-shots, scorecard, shot clock) Just Works for whoever is up; the per-shot POST swaps it at each end boundary.

- **WA set-system / cumulative scoring with a live scoreboard.** A combined scoreboard sits above the canvas: one row per archer with running total, X count, and — in set rounds — set points, with the active archer highlighted ("◀ shooting") and a "Now shooting: <name> (Archer k of N) · End n" banner. Scoring follows the round's `match_scoring` flag: **set system** (recurve/barebow, head-to-head) awards 2 set points to the higher end total, 1 each on a tie, and declares a winner at 6 set points — which can end the match before all five ends are shot; **cumulative** (compound) ranks by total once all ends are in. WA set play is only defined head-to-head, so for 3–4 archers the board falls back to a cumulative-total leaderboard. The "who's leading" line (e.g. "Hank leads 2–0 on set points") renders in Quantico italic.

- **Decided-match guard.** Once a set match is decided (an archer reaches 6 set points), the canvas input is disabled and the server refuses further shots, so a stale tap can't add an arrow past the win. The existing per-archer "round complete at full arrow count" guard still applies independently.

- **Match results carry the verdict.** The `/tournament/match/<id>` results page now shows the set-point / winner summary above the per-archer scorecards, ordered by seat. Ending a match (via End match / End round) closes every archer's session — not just the active one — and clears the live-match cookie keys.

- **CSRF tokens no longer expire after an hour.** Flask-WTF defaults `WTF_CSRF_TIME_LIMIT` to 3600 s, so a page left open longer than an hour — routine on a tablet mid-round, where the device sleeps between ends — failed its next POST with "CSRF token has expired" (surfacing as a 502 behind the proxy). Set `WTF_CSRF_TIME_LIMIT = None`: the token stays bound to the signed session cookie, so it remains valid only for the life of the login (30 days) and to that one session, but the one-hour age cap is gone. CSRF protection is otherwise unchanged — missing/forged tokens are still rejected.

- **Plumbing.**
  - New `match_scoring` field (`'set'` / `'cumulative'`) on the `wa_match_recurve_set` / `wa_match_recurve_cum` / `wa_match_compound` round defs — set-vs-cumulative is now data-driven instead of key-parsed.
  - Extracted `_mint_session_id(con, cur, user_id)` from the retry-with-backoff session-id loops duplicated in `tournament()` and `tournament_score_sheet_submit()`; reused by the new match-start route. `_owner_name_email()` similarly factored out of the scoresheet setup.
  - New `_match_set_scoring()` (set / cumulative / leaderboard), `_finalize_match_sessions()`, and routes `/tournament/match/setup` + `/tournament/match/start`. `tournament()` gained the shot-tagging (`match` / `participant` / `seat`), end-boundary archer swap, and scoreboard context; `match_id` / `match_archers` / `match_idx` added to every per-round session-cleanup pop list.
  - `_match_participants()` now parses the `seat:` tag so results order archers A/B/C/D.
