v0.52 — Analyze overhaul (continued): accuracy/precision split + five new reports

- **Accuracy vs precision, separated everywhere.** Every `/analyze` panel now reports the two error modes independently. Accuracy = MPI (‖centroid − bull‖). Precision = R95 about each group's *own* centroid (95% containment radius from the fitted bivariate normal; Rayleigh closed-form when isotropic, elliptical otherwise via χ²₂). Reports also surface signed bias (Δx, Δy) so you know which way to move the sight, plus Mean Radius, σ_x, σ_y, and extreme spread.
- **One shared stats helper.** `_archery_stats(xs, ys)` is the single spine: every report calls it. Samples below n=10 fall back to an empirical R95 percentile and the table flags that.
- **Shots per target — new overlays.** Concentric MR (solid) and R95 (dashed) rings about the centroid, a faint 1σ covariance ellipse exposing stringing, and a pink bias arrow from bull → centroid. Stats table split into Accuracy / Precision sections.
- **Accuracy & precision over time** (replaces the old "Accuracy over time"). Two clean traces — red MPI for accuracy, green R95 for precision. The old "σ from center" and "mean normalized distance" lines were hybrid signals that mixed bias and spread; they're gone. Per-bucket table adds MR, σ_x, σ_y.
- **Head-to-head — split verdicts.** Each pair now decomposes into three independent test families: **Accuracy** (Hotelling's T² on 2D centroids), **Precision** (Brown-Forsythe on distance-from-own-centroid, the honest spread signal — earlier versions fed it distance-from-bullseye, letting bias leak in), and **Total error** (Mann-Whitney U on distance-from-center, labelled as the hybrid it is). Holm-Bonferroni adjusts within each family. Verdict line now reads "Accuracy: … · Precision: … · Total error: …" so you can attribute a win to the right cause.
- **New report: Within-session drift.** Pools shots by quiver-index-in-session across all sessions, then plots MPI and R95 against that index. Reveals warm-up gains and late-session fatigue. Twin-axis grey bars show how many sessions contribute to each bucket so the user knows where confidence drops. Aggregates by quiver, not by arrow, since arrows aren't entered in shot order.
- **New report: Cold bore vs warmed up.** Pool 1 = first quiver of every session; pool 2 = quivers 2+. Side-by-side bars on MPI / R95 / pool size, then independent Hotelling T² (accuracy diff) and Brown-Forsythe (precision diff) with per-axis verdicts.
- **New report: Shot density heatmap.** Hexbin density overlay on each target with ≥25 hits, complementary to the scatter once a face accumulates hundreds of shots. Quadrant-breakdown table makes directional bias visible without reading the colorbar.
- **New report: Expected score from fit.** For each target with scoring zones, fits a 2D Gaussian to your shots, Monte-Carlo samples 20 000 draws (Cholesky-based correlated sampling), classifies each via the existing line-cutter rule, and projects expected points/arrow plus expected end scores at 3 / 6 / 10 arrows. Empirical sentinel-miss rate is blended in so total-flyers count against the projection.
- **New report: Shot volume calendar.** GitHub-style year-grid heatmap of shots per calendar day with month-boundary ticks. Summary table covers span, active-day %, busiest day, and avg shots on active days.
- **Hits by boundaries — face-colored bars.** Bars are sampled directly from the target image at each ring's outer edge instead of a synthetic warm-to-cool gradient. Works for any uploaded face (NASP, FITA, 3D, Vegas, NFAA). Falls back to the old gradient if the image can't be loaded.
- **Hovertips on every statistical name and metric.** Every test name (Hotelling, Brown-Forsythe, Mann-Whitney, Cliff's δ, Holm-Bonferroni) and every metric (MPI, R95, MR, σ_x/y, extreme spread, bias) carries a CSS tooltip explaining what it measures and why it's there. Tooltips render as wide-and-short rectangles (360 px width, ≈4:3 ratio) in light text on dark navy. Column-header tooltips drop below the cell instead of above to avoid the `.report-table-wrapper` overflow clip.
- **Imperial toggle wired into /analyze.** Length values render as `<span data-mm="…">33.4</span> <span class="metric-unit">mm</span>`. Click `⇄ Imperial` on the side nav and every length in every report converts to inches; click again to round-trip back. CSV / XLSX downloads always export raw mm regardless of toggle state. Units now display next to the value, not next to the metric name.
- **Per-report intro paragraphs.** Each new analytical report opens with a short framed explainer naming the question it answers — accuracy/precision colour-coded to match the chart traces (red for accuracy, green for precision).
- **CSV / XLSX export sanitizer.** Stripped Markup tooltip spans and HTML entities from exported cells so Excel sees clean plain text instead of raw HTML.
- **Bug fix: tooltip text colour.** Body-cell tooltips were inheriting black text on the navy background after an earlier edit moved the colour rule into a header-only override. Restored the single canonical rule so light text on dark navy applies everywhere.
