v0.59 — Fresh-clone fix: ship the bundled NASP seed image

- **A fresh `git clone` now works out of the box.** The default target seeded for every new user (`DEFAULT_TARGET_IMAGE`) and the placeholder image under every tournament face (`TOURNAMENT_PLACEHOLDER_IMAGE`) both point at `static/targets/nasp_40cm.jpg` — but `static/targets/` was fully gitignored, so the one asset the shipped faces depend on never made it into the repo. Anyone installing from a clean clone or `git archive` got a broken default target and broken tournament-face base images. The file was present on the original dev machine, so the breakage was invisible there and only surfaced on a fresh checkout.

- **The fix ships exactly that one image, nothing else.** Changed `.gitignore` from `static/targets/` to `static/targets/*` plus an explicit `!static/targets/nasp_40cm.jpg` un-ignore. (Git can't re-include a file whose *parent directory* is excluded, so the directory itself must stay tracked — only its contents are ignored.) All user-uploaded target images under `static/targets/` remain ignored as before; `nasp_40cm.jpg` is the sole tracked file there. No application code, paths, or the existing path-validation / `realpath`-under-`TARGETS_DIR` security checks changed — the image stays exactly where the runtime expects it.
